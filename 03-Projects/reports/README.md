# 03-Projects/reports README

Leicht strukturierte Report-Ablage:

- `audits/` — Audits, Reviews, Analysen, Coverage-Checks
- `runs/` — konkrete Run-/Tages-/Session-Reports
- `sprints/` — sprintbezogene Trigger, Endreports, Sprint-Reports

## Retrieval-Reihenfolge
1. Bei Diagnose-/Qualitätsfragen → `audits/`
2. Bei "was ist im Lauf passiert?" → `runs/`
3. Bei sprintbezogenen Verlaufsthemen → `sprints/`
4. Root nur für noch nicht einsortierte Restfälle
