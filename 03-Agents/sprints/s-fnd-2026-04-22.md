### T1 — Pydantic `SprintOutcome` Schema v1 definieren
**Context:** Research-C.1. Strikte Maschinen-Lesbarkeit der Task-Outcomes.
**Deliverable:** `src/schemas/sprint_outcome.py` (Pydantic v2) + JSONSchema export + 10 Fixture-Cases.
**Schema v1 Felder:**
- `schema_version: Literal['v1']`
- `status: done|partial|blocked|failed`
- `next_actions: list[NextAction{id, owner, priority, due, reason_code}]`

**DoD:** 10/10 Fixture-Cases validieren; p95 validation-latency <5ms; JSON-Schema geladen von MC-backend + Atlas-Runtime + Forge-Runtime.
**Feature-Flag:** `OUTCOME_SCHEMA_V1_ENABLED=0|1` (default 0, Folge-Sprints aktivieren)
**Rollback:** Delete schemas-module, kein Consumer bis Folge-Sprints.
**Owner:** Forge.

### T3 — Config-Integrity-Gate (validate→snapshot→apply→probe→revert)
**Context:** 2026-04-20 06:00 UTC Outage (25min) durch schema-invalid config-write. 3 Anwender: S-GOV, S-INFRA, S-INTEG.
**Deliverable:** `/home/piet/.openclaw/scripts/config-apply-safe.sh`:
```bash
config-apply-safe.sh <file> <new-content>
  1. Load JSONSchema for <file> (registry: /home/piet/.openclaw/schemas/)
  2. Validate new-content — exit 2 if invalid
  3. Snapshot: cp <file> <file>.bak-<ISO-timestamp>
  4. Apply: write new-content
  5. Probe: exec <file>.healthcheck (30s timeout) — exit 3 on fail
  6. On probe-fail: mv <file>.bak-<ts> <file>; exit 3
  7. Exit 0 + Discord alert "config applied"
```
**Schemas needed v1:** openclaw.json, crontab, registry.jsonl, agents.json, systemd-unit-drop-ins.
**DoD:**
- 3 Inject-Fault-Tests: schema-invalid/probe-fail/success → je korrekter Exit-Code + Snapshot
- 5 JSONSchemas committed

**Feature-Flag:** n/a (Tool)
**Rollback:** `rm config-apply-safe.sh`
**Owner:** Operator.

## Decision Log
- **Warum Pydantic statt nur TS-Interface?** Runtime-Validation am Receipt-Ingress blockt malformed Payloads, TS-Only prüft nur compile-time.
- **Warum attemptId sha256 statt UUID?** Deterministische Rekonstruktion für forensics.
- **Warum Config-Gate als Bash-Wrapper statt Python-CLI?** Bash ist überall, Python-Runtime kann self-block (Memory-Orchestrator-Lesson 2026-04-21).
- **Warum Jaeger statt Prometheus?** Trace-orientiert (was lief wann und wie lange) passt besser zu Sprint-Orchestration als Gauge-Metriken.
