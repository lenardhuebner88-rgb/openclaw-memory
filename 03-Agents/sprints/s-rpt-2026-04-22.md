# S-RPT: Reporting & Next-Action-Schema Hardening

## Why
Governance-Signale werden heute aus freien `resultSummary`/`resultDetails`-Strings abgeleitet (Regex/Keyword-Heuristik). Live-Evidenz: 136/270 Tasks mit überlangen narrative Feldern — Machine-Entscheidungen basieren auf Text-Match statt strukturierter State. P0.1 (Reader-Hygiene) + P0.2 (Pydantic Canonical Outcome) + Backfill = 3-Schichten-Modell: Human-Narrative / Canonical Outcome / Narrative Audit.

## Out-of-Scope
- P1.5 Follow-up als First-Class Object (→ Sprint-RPT2)
- P2 Lifecycle-Report-Sent Event (→ S-GOV OTEL)
- Cedar Policy-Engine Integration (→ S-GOV Spike)
