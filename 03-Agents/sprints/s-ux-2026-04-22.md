### T1 — Phase 0 stateTransitions[] im Backend (ROOT-CAUSE-FIX)
**Live-Status:** needed-new (Critic + Research-B)
**Deliverable:** `tasks.json` bekommt explizites Array:
```ts
stateTransitions: Array<{
  state: 'created' | 'dispatched' | 'pickup_claimed' | 'running' | 'failed' | 'done' | ...
  at: ISOTimestamp
  attemptId?: string  // ← S-FND T2
}>
```
Writer: Jeder state-change appended. Reader: Views lesen `stateTransitions.filter(s => s.state==='failed').at` statt updatedAt/lastActivityAt.

**DoD:**
- 10 synthetische Transitions: korrekt appended
- Reader-Query kann "failed since X" deterministisch beantworten
- `updatedAt` bleibt für API-ETag

**Feature-Flag:** `STATE_TRANSITIONS_V1=0|1` (dual-read)
**Rollback:** Flag off — Reader fällt auf alte Timestamps
**Owner:** Forge.
