# S-INTEG-W1: Claude Code × OpenClaw — Week 1 Foundation

## Why
Laptop (Windows, Claude Code) ↔ Homeserver (Linux, OpenClaw) hat 7 Friction-Punkte. 25min Outage 2026-04-20 durch Schema-invalide Config-Writes ohne Feedback-Loop. W1 schließt die 3 höchsten: Config-writes unaudited, No Read/Edit remote, No persistent memory-bridge. W2-W4 (Slash-Commands, Claude Design, Observability) als Follow-ups.

## Out-of-Scope (→ W2-W4)
- Slash-Commands `/sprint-dispatch`, `/atlas-status` (W2)
- Claude Design Handoff-Pipeline (W3)
- Dashboard Tooling-Health (W4)
- Custom SSH-Bridge (rejected)
- Full OTEL Bridge (→ S-GOV T8 MVP)

## Tasks

**Owner:** Operator.

### T3 — W1.3 vault-memory-MCP (minimal, nur für graph+rules)
**Live-Status:** needed (Research-Modify: custom **nur** für nicht-Standard-Tools)
**Deliverable:** Python-MCP-Server `/home/piet/.openclaw/mcp-servers/vault-memory/` mit **nur** Tools die kein Standard-Server hat:

**DoD:**

- Output-Schemas validieren

- Read-only mode 100% enforced

**Feature-Flag:** n/a (new server)
**Rollback:** Server stoppen + Config-Entry rm
**Owner:** Operator (Setup) + Forge (Schemas).

### T5 — W1.6 Schema-Gate-Hook (PostToolUse)
**Live-Status:** needed-new (Research-C + Critic: 2026-04-20 06:00 UTC Outage-Prevention)
**Deliverable:** `.claude/hooks/post-tool-use.sh`:
- Watches Write/Edit on `openclaw.json`, `crontab`, `registry.jsonl`, `agents.json`
- Calls S-FND T3 `config-apply-safe.sh` via SSH
- Blockt weiteren Tool-Call wenn exit != 0
- Fail-open nach 30s SSH-Timeout + Discord-Alert (Critic: fail-mode explizit!)

**DoD:**
- 3 Inject-Fault: schema-invalid/probe-fail/success → Hook-Behavior korrekt
- SSH-timeout-path: Discord-alert fires

**Feature-Flag:** `SCHEMA_GATE_HOOK=0|1` (default 0 bis Soak)
**Rollback:** Flag off + Hook entfernen
**Owner:** Forge.
