# OpenClaw Operational Folder

## Zweck
- Hier liegt die kompakte operative Wahrheit zum laufenden System.
- Fokus: Zustand, Betriebsregeln, technische Realität.

## Gehört hier rein
- `operational-state.md`
- kurze operative Zusammenfassungen
- knappe Systemzustände, die mehrere Agenten brauchen
- technische Wahrheiten, die nicht nur ein einzelnes Projekt betreffen

## Gehört nicht hier rein
- persönliche Präferenzen
- langfristige Projektplanung
- Audit-/Nachweis-Dokumente
- lose Recherchen

## Regel
- Wenn es dauerhaft für mehrere Agenten operativ wichtig ist, eher hier.
- Wenn es nur belegt, dass etwas geprüft wurde, nach `06-Operations/Validations`.
- Wenn es nur ein Agent akut braucht, eher in dessen `working-context.md`.
