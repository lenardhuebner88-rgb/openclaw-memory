## Cron Debug Session — Forge (23:00-00:00 UTC)

### Problem
8 cron jobs in ERROR state, all showing `LiveSessionModelSwitchError` or timeout errors.

### Root Causes Identified
1. **LiveSessionModelSwitchError**: Isolated cron sessions block model switches. Setting a different model than the agent's primary caused immediate failure.
2. **Timeout too short**: Complex tasks (nightly-self-improvement: 15-20min, weekly-pattern-detection: 3-5min) had timeouts too short for actual execution time.
3. **Ollama slowness**: session-cleanup-local using `ollama/qwen3.5:latest` through local-runner agent was too slow (>300s for a task that should take <1min).
4. **Task message too complex**: session-cleanup-local had an agentic prompt that caused long reasoning loops.

### Fixes Applied

| Cron | Fix |
|------|-----|
| model-compatibility-check | model → `minimax/MiniMax-M2.7-highspeed`, timeout → 150s |
| session-cleanup-local | Message simplified to direct bash commands, model → MiniMax (was Ollama qwen3.5), timeout → 180s |
| nightly-self-improvement | model → `minimax/MiniMax-M2.7-highspeed`, timeout → 600s |
| morning-kickoff | model → `minimax/MiniMax-M2.7-highspeed`, timeout → 180s |
| daily-cost-report | model → `minimax/MiniMax-M2.7-highspeed`, timeout → 120s |
| daily-standup-projekte | model → `openai-codex/gpt-5.4-mini` (already OK) |
| model-id-validator | model → `minimax/MiniMax-M2.7-highspeed`, timeout → 60s |
| weekly-pattern-detection | timeout → 300s (was 180s) |

### Key Insight
**Isolated sessions block model switches.** Cron jobs targeting isolated sessions must use the agent's primary model, or they fail with `LiveSessionModelSwitchError` immediately. Setting `payload.model` to the agent's own primary model is the correct approach.
